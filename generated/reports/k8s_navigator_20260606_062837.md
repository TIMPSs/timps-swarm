## Hotspots
- [CRITICAL] `Kubernetes API Server/API Server Connectivity` — kubectl cannot connect to the Kubernetes API server. (cause: The kubectl client is unable to reach the Kubernetes API server. This is likely due to an incorrect kubeconfig configuration, a down API server, or network connectivity issues. The error message 'localhost:8080' suggests kubectl is trying to connect to a default, unauthenticated local endpoint, which is rarely correct for a functional cluster.)

## Recommendations
### Verify kubectl configuration (kubeconfig)
The most common reason for 'connection refused' to localhost:8080 is an invalid or missing KUBECONFIG environment variable, or a corrupted/incorrect kubeconfig file. kubectl defaults to localhost:8080 if no valid configuration is found.
```bash
kubectl config view --minify
echo $KUBECONFIG
ls -la ~/.kube/config
```
### Check API Server status (if direct access to control plane is available)
If kubeconfig is correct, the API server itself might be down or unresponsive. This step requires SSH access to a control plane node.
```bash
ssh <control-plane-node-ip>
systemctl status kube-apiserver # For systemd-managed clusters
docker ps | grep kube-apiserver # For containerized control plane components
journalctl -u kube-apiserver.service # Check logs for API server
```
### Verify network connectivity to the API Server endpoint
Even with a correct kubeconfig, network issues (firewall, routing, VPN) can prevent kubectl from reaching the API server.
```bash
kubectl config view --minify | grep server
ping <api-server-host>
curl -k <api-server-url> # Use -k to ignore certificate errors initially, just check connectivity
```
